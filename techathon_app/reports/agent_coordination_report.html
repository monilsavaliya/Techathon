The "Baton Pass" Pattern

In traditional monolithic systems, massive data objects are passed from function to function. If one function crashes, the data is lost.

Our Solution: Pass ONLY the Reference ID.

Instead of passing megabytes of PDF data, the Main Agent (Orchestrator) simply passes the rfp_unique_id string (e.g., "RFP-2025-101").

Each agent independently Fetches the latest state, Processes its specific task, and Updates the central record directly. This ensures Zero Data Loss.

Modular Dependency Chain

The system enforces a Strict Dependency Chain. The Pricing Agent refuses to run if the Technical Agent hasn't marked its stage as "Completed".

This is enforced via the processing_stage_tracker block in the JSON.

  • Sales Agent: Parses text, creates the JSON skeleton.
  • Tech Agent: Reads Sales JSON, fills tech_output.
  • Pricing Agent: Reads Tech JSON, fills pricing_output.

This decoupling means you can Re-Run the Pricing Agent 10 times with different settings without re-parsing the PDF (Sales Agent) or re-matching products (Tech Agent). Highly Efficient.
